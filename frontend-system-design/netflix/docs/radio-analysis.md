## 1. Product Design vs System Design

### Key Difference

| Aspect | System Design | Product Design |
|--------|--------------|----------------|
| **Focus** | One key feature (infinite scroll, etc.) | Holistic view of entire application |
| **Depth** | Deep dive into algorithms | Shallow across all features |
| **Example** | Facebook News Feed virtualization | Netflix entire app structure |
| **Output** | Algorithm + data structure | Pages + components + data flow |

## 2. Requirements Exploration

### 2.1 General Requirements

| Feature | Description |
|---------|-------------|
| **Search** | By actor, genre, title |
| **Preview** | Short video preview on hover |
| **Watch** | Full movie/episode playback |
| **Browse** | Dashboard with movie lists by genre |
| **Promo** | Featured movie with hero section |

### 2.2 Functional Requirements

| Requirement | Description |
|-------------|-------------|
| **Multi-device** | TV, tablets, smartphones, desktops |
| **Adaptive Video** | Quality adapts to network bandwidth |
| **Optimized Media** | Images/promos optimized for viewport |
| **Offline Capable** | Cache resources with Service Worker |

### 7.3 Implementation

```typescript
// Actions
type NetflixAction =
  | { type: 'DASHBOARD_LOADED'; payload: DashboardResponse }
  | { type: 'MOVIE_LOADED'; payload: Movie }
  | { type: 'SEARCH_STARTED'; query: string }
  | { type: 'SEARCH_COMPLETED'; results: Movie[] }
  | { type: 'SET_CURRENT_MOVIE'; movieId: string | null };

// Reducer
function netflixReducer(state: NetflixStore, action: NetflixAction) {
  switch (action.type) {
    case 'DASHBOARD_LOADED': {
      const { promo, sections } = action.payload;
      const movies = new Map(state.movies);
      const dashboard = new Map<string, string[]>();
      
      // Normalize promo
      movies.set(promo.id, promo);
      
      // Normalize sections
      for (const [tag, movieList] of Object.entries(sections)) {
        const movieIds: string[] = [];
        for (const movie of movieList) {
          movies.set(movie.id, movie);
          movieIds.push(movie.id);
        }
        dashboard.set(tag, movieIds);
      }
      
      return {
        ...state,
        movies,
        dashboard,
        promo: promo.id,
      };
    }
    
    case 'MOVIE_LOADED': {
      const movies = new Map(state.movies);
      movies.set(action.payload.id, action.payload);
      return { ...state, movies };
    }
    
    // ... other cases
  }
}

// Selectors
function getDashboardMovies(state: NetflixStore, tag: string): Movie[] {
  const movieIds = state.dashboard.get(tag) || [];
  return movieIds.map(id => state.movies.get(id)).filter(Boolean) as Movie[];
}

function getMovieById(state: NetflixStore, id: string): Movie | undefined {
  return state.movies.get(id);
}
```

## Summary

| Section | Key Decision |
|---------|--------------|
| **Design System** | Shared components across teams |
| **Pages** | Sign Up (SSR) + Main (Feature-rich) |
| **Components** | Movie Card with 3 states (default, hover, full) |
| **Data** | Normalized store by ID |
| **Fetching** | Dashboard â†’ Lazy movie details |
| **Images** | CDN + Optimizer by viewport/bandwidth |
| **SSR** | Sign Up page for fast FCP |
| **Accessibility** | Hotkeys, subtitles, ARIA |

### Key Takeaways

1. **Product Design** - Holistic view, not deep dive on one feature
2. **Normalized Store** - O(1) access by ID, shared across views
3. **Lazy Loading** - Details only when needed (click)
4. **CDN Optimizer** - Serve right size for viewport/network
5. **SSR for Sign Up** - Critical for user conversion
6. **Preload + Cache** - Service Worker for offline support
